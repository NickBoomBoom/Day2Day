容器高塔：破解古代机关

一、任务目标

1. 掌握 Python 中列表（List）的创建、访问、修改、添加和删除元素的方法。

2. 理解元组（Tuple）的特性（有序、不可变）及其适用场景。

3. 学会使用字典（Dictionary）存储和访问键值对数据。

4. 通过破解古代机关的任务，实践不同容器类型在组织和管理数据中的应用。

5. 探索集合（Set）的特性（无序、唯一）及其在去重和成员检查中的妙用（彩蛋任务：集合魔法）。

你成功合成了第一个元素精灵，但要让它发挥真正的力量，需要从“容器高塔”中获取能量核心和控制指令。这座塔内部布满了精密的古代机关，每一层都由不同的数据结构守护。村长解释说：“这些机关的核心在于如何有效地存储和管理信息。列表像一串有序的珍珠，元组是永恒不变的符文序列，而字典则如同一个标签清晰的药材柜。”

代码精灵在你身边盘旋：“小魔法师，这次的挑战是驾驭数据的容器！列表灵活多变，元组稳定可靠，字典查找迅速。不同的机关需要不同的容器钥匙。准备好用正确的容器来破解它们了吗？”

三、任务实践

1. 知识准备

攀登容器高塔前，请熟悉这些容器魔法：

l   列表 (List) []: 有序的物品序列，可以随意添加、删除或更改其中的物品。如同一个魔法工具包，里面的工具可以随时更换。

索引访问: my_list[index]

修改: my_list[index] = new_value

添加: my_list.append(item)

删除: my_list.pop(index), my_list.remove(value)

l   元组 (Tuple) (): 有序的物品序列，一旦确定，就不能更改。如同刻在石板上的古老咒语，顺序和内容都固定不变。

索引访问: my_tuple[index]

不可修改

l   字典 (Dictionary) {} (键值对 key: value): 无序的标签集合，每个物品都有一个独特的标签（键），通过标签快速找到物品（值）。如同一个分类清晰的卷轴库。

键访问: my_dict[key]

添加/修改: my_dict[key] = new_value

获取所有键: my_dict.keys()

获取所有值: my_dict.values()

获取所有项: my_dict.items()

l   集合 (Set) {} (单个元素) 或 set(): 无序的、不重复的物品集合。如同一个自动去除重复护身符的魔法袋。

添加: my_set.add(item)

删除: my_set.remove(item)

成员检查: item in my_set (非常快)

2. 操作步骤

Step1 底层机关 - 序列齿轮 (列表 List)

故事情节:

塔的底层是一个由序列齿轮组成的机关。你需要按照正确的顺序激活齿轮，并且可能需要中途调整或增加步骤。代码精灵提示：“这种需要按顺序操作，并且可能随时修改序列的情况，列表是最合适的工具。”

任务:
(1) 记录初始序列: 创建一个列表 gear_sequence，包含需要按顺序激活的齿轮编号：[101, 102, 103, 104]。
(2) 检查第一个和最后一个: 打印出第一个和最后一个要激活的齿轮编号。
(3) 修正序列: 发现齿轮 103 需要更换为 103B。修改列表中的这个元素。
(4) 增加步骤: 在序列末尾增加一个新的齿轮 105。
(5) 移除错误步骤: 假设不小心添加了 999，需要将其移除。
(6) 打印最终序列: 显示调整后的完整激活序列。

示例代码:

# Step 1: 序列齿轮 (列表 List)

print("--- 底层机关：序列齿轮 ---")

gear_sequence = [101, 102, 103, 104]

print(f"初始齿轮序列: {gear_sequence}")

# 检查首尾

print(f"第一个齿轮: {gear_sequence[0]}")

print(f"最后一个齿轮: {gear_sequence[-1]}") # 负数索引从末尾开始

# 修正序列

gear_sequence[2] = "103B" # 索引从0开始，第三个元素索引是2

print(f"修正后序列: {gear_sequence}")

# 增加步骤

gear_sequence.append(105)

print(f"增加步骤后: {gear_sequence}")

# 假设添加了错误步骤 (先加进去再移除)

# gear_sequence.append(999)

# print(f"误加步骤后: {gear_sequence}")

# gear_sequence.remove(999)

# print(f"移除错误步骤后: {gear_sequence}")

# 或者如果知道位置可以用 pop: gear_sequence.pop(-1) # 移除最后一个

print(f"最终激活序列: {gear_sequence}")

思考点: 留意列表是如何通过索引访问和修改元素的，以及 append 如何在末尾添加新元素。列表的灵活性是其特点。

Step2 中层枢纽 - 符文石板 (元组 Tuple)

故事情节:

来到塔的中层，你面对一块刻有符文的石板。这些符文代表了启动中枢能量的固定密码，顺序和内容绝不能改变。代码精灵提醒：“对于这种一旦设定就不允许更改的固定序列，元组是更安全的选择。”

任务:
(1) 定义符文密码: 创建一个元组 rune_password，包含符文序列：("Alpha", "Beta", "Gamma")。
(2) 读取密码: 打印出密码的第一个符文和整个密码序列。
(3) 尝试修改 (并观察结果): 尝试将第二个符文 "Beta" 修改为 "Omega"。观察 Python 返回的错误信息。

列表和元组看起来很像，都能存储有序的元素。但元组的“不可变性”是关键区别。为什么需要这种不可变的特性？它有什么好处？如果你对此感到好奇，可以请教你的代码精灵：“Python 中元组（Tuple）相比列表（List）的主要优点是什么？在什么情况下应该优先选用元组？” 探索其在安全性、性能以及作为字典键等方面的用途。如图1所示。

示例代码:

# Step 2: 符文石板 (元组 Tuple)

rune_password = ("Alpha", "Beta", "Gamma")

print(f"符文密码序列: {rune_password}")

print(f"第一个符文: {rune_password[0]}")

# 尝试修改元组 (预期会失败)

print("尝试修改第二个符文...")

try:

    rune_password[1] = "Omega"

except TypeError as e:

    print(f"修改失败！错误信息: {e}")

Step3 高层核心 - 零件仓库 (字典 Dictionary)

故事情节:

接近塔顶，你需要组装一个复杂的能量核心。核心的零件存储在一个大型仓库里，每个零件都有唯一的名称标签和对应的库存数量。代码精灵指出：“按名字查找物品，而不是按顺序位置，字典是最佳选择。”

任务:
(1) 建立库存记录: 创建一个字典 component_inventory，记录零件及其数量：{"核心水晶": 5, "能量导管": 12, "稳定器": 8}。
(2) 查询库存: 打印出 "能量导管" 的库存数量。
(3) 添加新零件: 仓库新增了 "冷却液"，数量为 20 瓶。将其添加到字典中。
(4) 更新库存: 消耗了 2 个 "核心水晶"，更新其库存数量。
(5) 展示所有零件及其库存: 遍历字典，打印出所有零件名称和对应的数量。

字典通过“键”（Key）来快速定位“值”（Value），这与列表/元组通过数字索引访问的方式截然不同。这种键值对的结构在什么场景下特别有用？它和现实生活中的哪些例子相似？如果你想深入理解字典的查找效率为何通常很高（即使数据量很大），可以向 AIGC 请教：“简单解释一下 Python 字典是如何通过键快速找到值的（哈希表原理简介）？”

图2 如何通过键快速找到值

示例代码:

# Step 3: 零件仓库 (字典 Dictionary)

component_inventory = {"核心水晶": 5, "能量导管": 12, "稳定器": 8}

print(f"初始库存: {component_inventory}")

# 查询库存

part_name = "能量导管"

print(f"查询 \'{part_name}\' 库存: {component_inventory[part_name]} 个")

# 添加新零件

component_inventory["冷却液"] = 20

print(f"添加 \'冷却液\' 后库存: {component_inventory}")

# 更新库存

component_inventory["核心水晶"] -= 2 # 或者 component_inventory["核心水晶"] = component_inventory["核心水晶"] - 2

print(f"更新 \'核心水晶\' 后库存: {component_inventory}")

# 展示所有零件及库存

print("当前仓库所有零件及库存:")

for part, quantity in component_inventory.items(): # 使用 .items() 获取键值对

    print(f"- {part}: {quantity} 个")

Step4 彩蛋探索 - 净化祭坛 (集合 Set)

故事情节:

在塔顶，你发现一个古老的净化祭坛。要激活它，需要献上纯净的能量符印。你收集了一袋符印，但里面可能混有重复的。祭坛只会接受每种符印各一个。代码精灵耳语：“集合魔法擅长处理独一无二的物品！”

任务:

(1) 收集的符印 (含重复): 创建一个列表 collected_sigils，包含一些重复的符印名称：["火", "水", "风", "土", "水", "火", "光"]。

(2) 净化符印 (去重): 使用 set() 将列表转换为集合，得到独一无二的符印集合 pure_sigils。打印这个集合，观察重复项是否消失。

(3) 检查特定符印: 检查 "暗" 符印是否存在于 pure_sigils 中。再检查 "风" 符印是否存在。

(4) 添加新发现的符印: 你又找到了一个 "雷" 符印，将其添加到 pure_sigils 集合中。打印最终的纯净符印集合。

示例代码:

# Step 4: 净化祭坛 (集合 Set)

print("\\n--- 彩蛋：净化祭坛 ---")

collected_sigils = ["火", "水", "风", "土", "水", "火", "光"]

print(f"收集到的符印 (含重复): {collected_sigils}")

# 净化符印 (去重)

pure_sigils = set(collected_sigils)

print(f"净化后的纯净符印 (集合): {pure_sigils}") # 注意输出顺序可能不同，且无重复

# 检查特定符印

print(f"\'暗\' 符印是否存在? {\'暗\' in pure_sigils}")

print(f"\'风\' 符印是否存在? {\'风\' in pure_sigils}")

# 添加新符印

pure_sigils.add("雷")

print(f"添加 \'雷\' 符印后: {pure_sigils}")

思考点: 集合最显著的特点是元素的唯一性和无序性。思考一下，除了去重，快速判断一个元素是否存在于大量数据中，集合为什么通常比列表更快？（提示：这和字典的快速查找原理类似）。
